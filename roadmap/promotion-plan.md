# План продвижения GenesysForge

Цель: привести игроков и мастеров Genesys / Realms of Terrinoth на https://genesys-forge.com и превратить их в постоянных пользователей.

Формат: каждый пункт — самостоятельная задача для ИИ-агента (Claude Code / Codex). У задачи есть **Вход** (что нужно знать/иметь), **Шаги** и **Готово, когда**. Пометка `[HUMAN]` — шаг требует человека (создание аккаунтов, финальная публикация от своего имени, оплата). Агент в таких задачах готовит материал «под публикацию», человек нажимает кнопку.

Принцип приоритезации: аудитория Genesys маленькая и концентрированная, поэтому нишевые сообщества и каталоги дают больше, чем массовый SEO. Порядок фаз — по соотношению эффект/усилия.

## Фаза 0 — измерение (без него нельзя понять, что сработало)

### 0.1 Подключить аналитику без cookie-баннера
- **Вход:** доступ к VPS (Beget, 46.173.17.211), docker-compose.prod.yml.
- **Шаги:** развернуть self-hosted [Umami](https://umami.is) или Plausible CE рядом с Gatus в /opt/monitoring (или отдельным compose-сервисом); добавить скрипт в `frontend/index.html`; вынести дашборд на поддомен (например, `stats.genesys-forge.com`, wildcard origin cert уже есть).
- **Готово, когда:** в дашборде видны визиты и источники переходов (referrers), UTM-метки учитываются.

### 0.2 Зарегистрировать сайт в поисковиках
- **Вход:** `[HUMAN]` Google-аккаунт и Яндекс-аккаунт владельца.
- **Шаги:** Google Search Console + Yandex Webmaster: подтвердить домен (DNS TXT через Cloudflare), отправить sitemap.xml; проверить, что robots.txt отдаётся корректно (после деплоя SEO-правок).
- **Готово, когда:** обе консоли показывают «sitemap принят», страницы `/`, `/about`, `/help` проиндексированы.

### 0.3 UTM-дисциплина
- **Шаги:** для каждой публикации из фаз 1–4 использовать ссылку вида `https://genesys-forge.com/?utm_source=<канал>&utm_medium=post&utm_campaign=launch`. Агент, готовящий пост, обязан вставить такую ссылку.
- **Готово, когда:** правило записано в этом файле и соблюдается во всех черновиках.

## Фаза 1 — точки, где уже сидит аудитория Genesys (максимальный эффект)

### 1.1 Пост в r/genesysrpg (≈8 тыс. подписчиков, все — целевые)
- **Вход:** скриншоты интерфейса (лист, пирамида талантов, game table), список фич из README. `[HUMAN]` аккаунт Reddit с небольшой историей.
- **Шаги (агент):** написать пост на английском в жанре «I built a free web character sheet for Genesys & Realms of Terrinoth» — честный тон разработчика, 5–7 фич, ссылка с UTM, приглашение к фидбеку; подготовить 3–4 скриншота (браузером: открыть сайт, resize 1280×800, screenshot). Проверить правила сабреддита на self-promotion.
- **Шаги `[HUMAN]`:** опубликовать, первые сутки отвечать на комментарии (агент может готовить черновики ответов).
- **Готово, когда:** пост опубликован, ответы на все комментарии первых 48 часов даны.

### 1.2 Discord-сообщества Genesys
- **Вход:** `[HUMAN]` аккаунт Discord. Ключевые серверы: официальный Genesys RPG Community, Terrinoth-сообщества, крупные русскоязычные НРИ-серверы.

- **Шаги `[HUMAN]`:** вступить, освоиться пару дней, запостить в разрешённых каналах.
- **Готово, когда:** сообщение размещено минимум в 3 серверах без нарушения правил.

### 1.3 Русскоязычные НРИ-сообщества (родная аудитория проекта)
- **Вход:** `[HUMAN]` аккаунты VK/Telegram.
- **Шаги (агент):** подготовить по одному посту на русском для: VK-групп по НРИ и Genesys, Telegram-каналов/чатов НРИ (составить актуальный список через веб-поиск: «Genesys НРИ telegram», «настольные ролевые игры чат»), форума imaginaria.ru и rpg-world.org. Тон — «сделал бесплатный инструмент для нашего хобби, буду рад фидбеку». Отдельно упомянуть Realms of Terrinoth (по нему мало инструментов вообще).
- **Готово, когда:** черновики лежат в `roadmap/promotion-drafts/ru-communities.md`, человек опубликовал минимум в 3 местах.

### 1.4 Каталоги ресурсов Genesys
- **Шаги (агент):** найти и оформить заявки/PR: awesome-списки на GitHub («awesome ttrpg», «awesome genesys»), вики и pinned-ресурсы r/genesysrpg, каталоги бесплатных TTRPG-инструментов (itch.io tools, ttrpg-tools списки). Для GitHub-списков — сразу готовый PR с однострочным описанием на английском.
- **Готово, когда:** отправлено ≥5 заявок/PR, ссылки зафиксированы в `roadmap/promotion-log.md`.

## Фаза 3 — SEO-контент (медленный, но накапливающийся канал)

### 3.1 Публичные страницы-гайды
- **Вход:** сейчас без логина доступны только `/`, `/about`, `/help` — краулерам почти нечего индексировать.
- **Шаги (агент):** добавить публичный раздел `/guides` (SPA-маршруты + пункты в sitemap.xml): «Как собрать дайс-пул в Genesys», «Пирамида талантов: правила покупки», «Genesys vs Realms of Terrinoth: отличия», «Как провести первую сессию Genesys». Каждая страница: H1, 700+ слов, внутренние ссылки на приложение, JSON-LD Article. Обновить sitemap. Это одновременно и справка для пользователей.
- **Готово, когда:** ≥4 гайда в проде, добавлены в sitemap, Search Console видит их без ошибок.

### 3.2 Доступность для ИИ-ассистентов (AEO)
- **Шаги (агент):** добавить `frontend/public/llms.txt` с кратким описанием сервиса, возможностей и публичных URL; убедиться, что robots.txt не блокирует GPTBot/ClaudeBot/PerplexityBot. Люди всё чаще спрашивают инструменты у ассистентов — сайт должен быть «читаем» для них (noscript-контент из SEO-правок уже помогает).
- **Готово, когда:** `https://genesys-forge.com/llms.txt` отдаётся с корректным содержимым.

### 3.3 Prerender публичных страниц (по желанию, после 3.1)
- **Шаги (агент):** на этапе `vite build` пререндерить `/`, `/about`, `/help`, `/guides/*` в статический HTML (vite-plugin-prerender или самописный скрипт с Playwright, который уже в devDependencies), чтобы краулеры получали контент без JS.
- **Готово, когда:** `curl https://genesys-forge.com/about` возвращает HTML с текстом страницы.
