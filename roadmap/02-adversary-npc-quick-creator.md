# Adversary / NPC Quick Creator

## 1. Назначение

**Adversary / NPC Quick Creator** — инструмент для быстрого создания противников, союзников и NPC для Genesys / Realms of Terrinoth.

Мастер должен за 1–2 минуты создать пригодную для игры карточку:

- миньона;
- ривала;
- немезиды;
- социального NPC;
- монстра;
- мага;
- босса;
- союзника;
- группы врагов.

Фича не должна импортировать или копировать официальные stat blocks из книг. Она должна создавать пользовательские NPC и generic drafts.

---

## 2. Пользовательская ценность

### Для мастера

Мастер может:

- быстро подготовить противников;
- хранить NPC в библиотеке;
- фильтровать NPC по типу, роли и тегам;
- дублировать и изменять NPC;
- добавлять NPC в Encounter Builder;
- добавлять NPC в Game Table;
- печатать карточки.

### Для игрока

Игрок может видеть только публичные NPC кампании, если мастер их открыл.

---

## 3. Типы adversary

## 3.1. Minion

Слабый противник, часто используется группами.

Особенности:

- обычно не имеет strain threshold;
- может входить в minion group;
- имеет упрощённую карточку;
- подходит для массовых врагов.

## 3.2. Rival

Средний NPC.

Особенности:

- важнее миньона;
- может иметь навыки, способности и снаряжение;
- используется как командир, социальный оппонент, опасный враг или союзник.

## 3.3. Nemesis

Главный NPC или босс.

Особенности:

- имеет strain threshold;
- имеет более полную карточку;
- подходит для важных злодеев, магов, монстров и лидеров.

---

## 4. Роли NPC

Роль помогает быстро создавать черновики и фильтровать библиотеку.

Базовые роли:

- **Brute** — сильный ближник с высоким Brawn и Soak;
- **Skirmisher** — быстрый мобильный противник;
- **Archer** — дальний атакующий;
- **Caster** — маг или мистик;
- **Leader** — командир, усиливающий других;
- **Social** — переговорщик, интриган, политик;
- **Support** — лекарь, баффер, помощник;
- **Monster** — зверь или чудовище;
- **Custom** — произвольная роль.

## 6. Основные сценарии

## 6.1. Создание NPC вручную

1. Мастер открывает **NPC / Бестиарий**.
2. Нажимает **Создать NPC**.
3. Выбирает систему, тип, роль и имя.
4. Заполняет характеристики.
5. Заполняет derived stats.
6. Добавляет навыки.
7. Добавляет способности, таланты и снаряжение.
8. Сохраняет NPC.

## 6.2. Быстрый черновик

1. Мастер нажимает **Быстрый черновик**.
2. Выбирает:
   - систему;
   - тип;
   - роль;
   - уровень силы;
   - основную характеристику;
   - боевой стиль.
3. Система генерирует generic draft.
4. Мастер редактирует результат.
5. Сохраняет NPC.

Уровни силы:

- weak;
- standard;
- strong;
- elite.

Генератор должен быть deterministic, без LLM в MVP.

## 6.3. Дублирование NPC

1. Мастер открывает NPC.
2. Нажимает **Дублировать**.
3. Система создаёт копию.
4. Мастер меняет имя и параметры.
5. Сохраняет.

## 6.4. Добавление в Game Table

1. Мастер открывает NPC.
2. Нажимает **Добавить в сцену**.
3. Выбирает активную кампанию/сцену.
4. Указывает количество и формат: одиночные NPC или minion group.
5. Система создаёт GameTableParticipant.

## 6.5. Добавление в Encounter Builder

1. Мастер открывает Encounter.
2. Нажимает **Добавить NPC**.
3. Выбирает NPC из библиотеки.
4. Указывает количество.
5. NPC добавляется в EncounterParticipant.

---

## 7. Библиотека NPC

Страница должна поддерживать:

- поиск по имени;
- фильтр по системе;
- фильтр по типу;
- фильтр по роли;
- фильтр по тегам;
- фильтр по кампании;
- сортировку по имени;
- сортировку по дате создания;
- избранное.

Карточка NPC в списке показывает:

- имя;
- тип;
- роль;
- system;
- soak;
- wound threshold;
- главные навыки;
- теги;
- действия.

## 8. Доступы

| Действие | GM | Игрок | Не участник |
|---|---:|---:|---:|
| Создать NPC | Да | Нет в MVP | Нет |
| Редактировать свой NPC | Да | Нет | Нет |
| Удалить свой NPC | Да | Нет | Нет |
| Дублировать NPC | Да | Нет | Нет |
| Видеть приватного NPC | Только владелец | Нет | Нет |
| Видеть NPC кампании | Да | Да, если visible | Нет |
| Добавить NPC в сцену | Да | Нет | Нет |
| Добавить NPC в энкаунтер | Да | Нет | Нет |
| Печатать NPC | Да | Только visible версию | Нет |

---

## 9. Валидация

Правила:

- имя обязательно;
- характеристики от 1 до 6;
- ranks навыков от 0 до 5;
- wounds threshold больше 0;
- soak не может быть отрицательным;
- defense не может быть отрицательной;
- Minion обычно не имеет strain threshold;
- Nemesis должен иметь strain threshold.

---

## 10. MVP

В MVP входит:

- CRUD NPC;
- Minion / Rival / Nemesis;
- ручное создание;
- быстрый deterministic draft;
- библиотека NPC;
- фильтры;
- дублирование;
- добавление в Game Table;
- базовая печатная карточка.

---

## 11. Не входит в MVP

- импорт официальных NPC из книг;
- AI-генерация;
- marketplace;
- публичная библиотека сообщества;
- автоматический баланс энкаунтера;
- сложные шаблоны поведения.

---

## 12. Критерии готовности

Фича готова, если:

- мастер может создать NPC вручную;
- мастер может создать NPC через быстрый черновик;
- NPC сохраняется в библиотеке;
- NPC можно найти через поиск/фильтр;
- NPC можно добавить в Game Table;
- NPC можно дублировать;
- игрок не видит приватных NPC;
- нет копирования официальных stat blocks.
