# GenesysForge — пробелы по аудиту (`GENESYSFORGE_AUDIT.md`)

Документ фиксирует **расхождения между требованиями аудита и фактическим кодом** на дату аудита
(2026-06-24, ветка `master`). Это вход для общего плана — см. [unified-roadmap.md](unified-roadmap.md).

Легенда: ✅ есть · ⚠️ частично · ❌ нет. Указаны только пункты со статусом ⚠️/❌
(полностью реализованные опущены — их перечень в исходном аудите).

> Фактический стек (важно, в аудите указан Node-псевдокод): **C#/.NET 10 Minimal API
> (Clean Architecture + CQRS) + React 19/TS**, PostgreSQL/EF Core, SignalR, Docker/Caddy.
