# GM Cockpit / Game Table

## 1. Назначение

**GM Cockpit / Game Table** — экран мастера для ведения текущей сцены в кампании Genesys / Realms of Terrinoth. Это не полноценный VTT с картой и токенами, а компактный пульт управления сценой: участники, инициатива, story points, wounds/strain, заметки, статусы и текущий раунд.

Главная цель: мастер должен вести сцену из одного окна, не переключаясь между листами персонажей, заметками, таблицами и сторонними трекерами.

---

## 2. Пользовательская ценность

### Для мастера

Мастер может:

- создать активную сцену кампании;
- добавить персонажей игроков;
- добавить NPC и группы миньонов;
- управлять story points;
- вести инициативу по слотам Genesys;
- отслеживать wounds, strain, soak и defense;
- скрывать часть информации от игроков;
- вести публичные и приватные заметки сцены;
- завершать или сбрасывать сцену.

### Для игрока

Игрок может:

- видеть текущую сцену;
- видеть текущий раунд и слот инициативы;
- видеть story points;
- видеть своего персонажа и публичных участников;
- при разрешении мастера менять wounds/strain своего персонажа.

## 4. Основные сущности

## 4.1. GameTableState / CampaignSession

Состояние активной сцены кампании.

Поля:

| Поле | Описание |
|---|---|
| Id | Идентификатор сцены |
| CampaignId | Кампания |
| Name | Название сцены |
| Description | Описание сцены |
| IsActive | Активна ли сцена |
| PlayerStoryPoints | Story points игроков |
| GmStoryPoints | Story points мастера |
| CurrentRound | Текущий раунд |
| CurrentTurnIndex | Индекс текущего слота инициативы |
| PublicNotes | Заметки, видимые игрокам |
| GmNotes | Приватные заметки мастера |
| CreatedAt | Дата создания |
| UpdatedAt | Дата обновления |

## 4.2. GameTableParticipant

Участник сцены.

Поля:

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| SessionId | Активная сцена |
| CharacterId | Персонаж игрока, если это PC |
| AdversaryDefId | NPC из библиотеки, если есть |
| DisplayName | Имя на экране |
| ParticipantType | PlayerCharacter / Npc / MinionGroup / Hazard |
| InitiativeSlotType | Player / Npc / Neutral |
| WoundsCurrent | Текущие раны |
| WoundsThreshold | Порог ран |
| StrainCurrent | Текущая усталость |
| StrainThreshold | Порог усталости |
| Soak | Поглощение |
| MeleeDefense | Ближняя защита |
| RangedDefense | Дальняя защита |
| IsActive | Активен ли участник |
| IsDefeated | Побеждён ли участник |
| IsHiddenFromPlayers | Скрыт ли от игроков |
| Notes | Заметки мастера |

## 4.3. InitiativeSlot

Слот инициативы Genesys.

Поля:

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| SessionId | Сцена |
| SlotType | Player / Npc |
| Order | Порядок |
| AssignedParticipantId | Назначенный участник, необязательно |
| Notes | Комментарий |

## 5. Пользовательские сценарии

## 5.1. Создание сцены

1. GM открывает кампанию.
2. Переходит во вкладку **Game Table**.
3. Если активной сцены нет, нажимает **Создать сцену**.
4. Заполняет название, описание и начальные story points.
5. Система создаёт активную сцену.
6. GM видит пустой Game Table и может добавлять участников.

## 5.2. Добавление участников

GM может добавить:

- всех персонажей кампании;
- выбранных персонажей;
- NPC из Adversary Library;
- временного NPC вручную;
- minion group.

При добавлении из персонажа или NPC система копирует базовые параметры: wounds, strain, soak, defense, display name.

## 5.3. Управление инициативой

GM может:

- добавить Player Slot;
- добавить NPC Slot;
- изменить порядок слотов;
- удалить слот;
- назначить участника на слот;
- оставить слот абстрактным.

Кнопка **Следующий ход** переключает CurrentTurnIndex. После последнего слота начинается новый раунд.

## 5.4. Story Points

На экране есть два счётчика:

- Players;
- GM.

Функции:

- добавить игрокам;
- убрать у игроков;
- добавить мастеру;
- убрать у мастера;
- перевернуть от игроков к GM;
- перевернуть от GM к игрокам.

Story points не могут быть отрицательными.

## 5.5. Изменение состояния участника

GM может:

- увеличить/уменьшить wounds;
- увеличить/уменьшить strain;
- пометить участника defeated;
- вернуть участника active;
- скрыть/показать участника игрокам;
- изменить заметки;
- удалить участника из сцены.

## 5.6. Просмотр игроком

Игрок видит:

- название сцены;
- текущий раунд;
- текущий слот инициативы;
- story points;
- своего персонажа;
- публичных участников;
- публичные заметки.

Игрок не видит:

- hidden NPC;
- GM notes;
- скрытые осложнения;
- приватные статусы.

## 6. UI

## 6.1. Структура страницы

В CampaignDetailView добавить вкладку **Game Table**.

Основные блоки:

1. Header сцены.
2. Story Points.
3. Initiative Tracker.
4. Participants.
5. Notes.
6. GM actions.

## 6.2. Header сцены

Показывает:

- название;
- статус active;
- текущий раунд;
- текущий ход;
- кнопки: **Следующий ход**, **Завершить**, **Сбросить**.

## 6.3. Participants

Карточка участника показывает:

- имя;
- тип;
- wounds;
- strain;
- soak;
- defense;
- defeated state;
- visibility;
- notes.

---

## 7. MVP

В MVP входит:

- одна активная сцена на кампанию;
- создание/сброс/завершение сцены;
- story points;
- участники;
- wounds/strain;
- initiative slots;
- next turn;
- player read-only view;
- GM-only управление.

---

## 8. Не входит в MVP

- WebSocket real-time;
- карта;
- токены;
- drag-and-drop;
- автоматические броски инициативы;
- сложные статусы;
- история всех сцен;
- PDF-экспорт.

---

## 9. Критерии готовности

Фича готова, если:

- GM может создать активную сцену;
- GM может добавить PC и NPC;
- GM может управлять story points;
- GM может вести инициативу;
- игрок кампании видит сцену;
- не участник кампании не имеет доступа;
- приватные заметки не видны игрокам;
- данные сохраняются после перезагрузки.
