# Magic / Alchemy / Rune Assistant

## 1. Назначение

**Magic / Alchemy / Rune Assistant** — набор инструментов для Realms of Terrinoth и Genesys, который помогает собирать магические действия, алхимические рецепты и рунические улучшения.

Фича решает практическую проблему:

> “Мне нужно быстро понять сложность действия, выбранные дополнительные эффекты, траты преимуществ/угроз и сохранить результат как карточку.”

Модуль состоит из трёх частей:

1. Magic Action Builder.
2. Alchemy Assistant.
3. Rune Assistant.

Первым MVP рекомендуется делать **Magic Action Builder**, потому что он лучше всего связан с уже существующим справочником spells.

---

# Часть 1. Magic Action Builder

## 2. Суть

Magic Action Builder позволяет пользователю:

- выбрать систему;
- выбрать магический навык;
- выбрать базовый эффект;
- выбрать дополнительные эффекты;
- увидеть итоговую сложность;
- увидеть доступные траты advantage/threat/triumph/despair;
- сохранить или скопировать карточку.

---

## 3. Пользователи

### GM

Может:

- использовать builder без персонажа;
- создавать кастомные магические эффекты;
- добавлять эффекты в Campaign Handbook;
- видеть GM-only заметки;
- использовать результат в энкаунтере или Game Table.

### Player

Может:

- использовать builder для своего персонажа;
- выбирать свои magic skills;
- собирать действие;
- копировать результат;
- сохранять личные шаблоны.

---

| CostThreat | Стоимость в Threat |

---

## 5. Основные сценарии Magic Builder

## 5.1. Игрок собирает заклинание

1. Игрок открывает персонажа.
2. Нажимает **Создать магическое действие** рядом с магическим навыком.
3. Открывается Magic Builder.
4. Персонаж и навык уже выбраны.
5. Игрок выбирает базовое действие.
6. Игрок выбирает дополнительные эффекты.
7. Система показывает dice pool, базовую сложность, итоговую сложность, выбранные эффекты и траты преимуществ/угроз.
8. Игрок копирует результат или сохраняет как шаблон.

## 5.2. GM собирает магическое действие без персонажа

1. GM открывает вкладку **Магия**.
2. Выбирает систему.
3. Выбирает magic skill.
4. Собирает действие.
5. Использует результат как NPC spell или эффект сцены.

## 5.3. Создание кастомного магического эффекта

1. Пользователь нажимает **Создать кастомный эффект**.
2. Заполняет название, system, magic skill, difficulty, safe summary, tags.
3. Сохраняет.
4. Эффект появляется в личной библиотеке.

---

## 6. Функции Magic Builder

- выбор системы;
- выбор персонажа;
- выбор magic skill;
- фильтр базовых действий;
- выбор дополнительных эффектов;
- расчёт сложности;
- предупреждения;
- копирование результата;
- сохранение шаблона;
- добавление результата в заметку;
- печатная карточка.

---

## 7. Валидация Magic Builder

Правила:

- эффект должен принадлежать выбранной системе;
- additional effect должен быть совместим с базовым действием;
- total difficulty не может быть меньше base difficulty;
- custom modifiers должны быть в разумном диапазоне;
- official effects не должны содержать длинный текст из книги;
- source/page ref обязательны для official effects.

# Часть 2. Alchemy Assistant

## 8. Суть

Alchemy Assistant помогает создавать и вести алхимические рецепты, ингредиенты и готовые зелья.

Он нужен для:

- крафта зелий;
- поиска ингредиентов;
- оценки стоимости;
- расчёта сложности;
- создания предмета в инвентаре после успешного крафта.

---

## 9. Сущности Alchemy

## 9.1. AlchemyRecipe

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| OwnerUserId | Владелец |
| CampaignId | Кампания |
| Name | Название |
| Effect | Эффект |
| Difficulty | Сложность создания |
| Ingredients | Ингредиенты |
| Cost | Стоимость |
| CraftTime | Время создания |
| Rarity | Редкость |
| SafeSummary | Краткое описание |
| Source | Источник |
| PageRef | Страница |
| IsCustom | Кастомный рецепт |

## 9.2. AlchemyIngredient

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| Name | Название |
| Rarity | Редкость |
| Cost | Стоимость |
| Region | Регион |
| IsLegal | Легальность |
| Description | Описание |
| CampaignAvailability | Доступность в кампании |

---

## 10. Сценарий крафта

1. Игрок выбирает рецепт.
2. Система показывает ингредиенты, стоимость, сложность, время и ограничения.
3. Игрок отмечает доступные ингредиенты.
4. Система формирует чек крафта.
5. GM подтверждает успех.
6. Система создаёт предмет или предлагает добавить его в инвентарь.

---

## 11. Доступы Alchemy

| Действие | GM | Player |
|---|---:|---:|
| Создать рецепт кампании | Да | Нет |
| Создать личный рецепт | Да | Да |
| Видеть разрешённые рецепты | Да | Да |
| Добавить ингредиент кампании | Да | Нет |
| Создать предмет после крафта | Да | Да, если разрешено |
| Подтвердить результат крафта | Да | Нет |

---

# Часть 3. Rune Assistant

## 12. Суть

Rune Assistant помогает создавать руны, применять их к предметам и сохранять зачарованные предметы.

Фича особенно полезна для Realms of Terrinoth.

---

## 13. Сущности Rune

## 13.1. RuneDef

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| Name | Название |
| System | Система |
| Effect | Эффект |
| Difficulty | Сложность |
| Cost | Стоимость |
| Restrictions | Ограничения |
| SafeSummary | Краткое описание |
| Source | Источник |
| PageRef | Страница |
| IsCustom | Кастомная руна |

## 13.2. ItemRuneApplication

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| CharacterItemId | Предмет персонажа |
| RuneDefId | Руна |
| AppliedByUserId | Кто применил |
| ApprovedByGm | Подтверждено ли GM |
| Notes | Заметки |
| CreatedAt | Дата |

---

## 14. Сценарий применения руны

1. Игрок открывает Rune Assistant.
2. Выбирает предмет из инвентаря.
3. Выбирает руну.
4. Система показывает требования, сложность, стоимость, ограничения и итоговый эффект.
5. Игрок отправляет на подтверждение GM.
6. GM подтверждает.
7. Система создаёт enchanted item или добавляет rune slot к предмету.

---

## 15. Доступы Rune Assistant

| Действие | GM | Player |
|---|---:|---:|
| Создать руну кампании | Да | Нет |
| Создать личную кастомную руну | Да | Да |
| Применить руну к своему предмету | Да | Да, с подтверждением |
| Подтвердить применение руны | Да | Нет |
| Видеть GM-only ограничения | Да | Нет |

---

## 16. MVP

В первый MVP входит только Magic Action Builder:

- базовые magic actions;
- additional effects;
- расчёт сложности;
- character integration;
- copy result;
- safe source/page refs.

Alchemy и Rune Assistant можно делать вторым этапом.

---

## 17. Не входит в MVP

- полный официальный текст правил;
- AI-генерация заклинаний;
- автоматическая проверка всех исключений;
- сложные crafting chains;
- marketplace рецептов;
- PDF-экспорт.

---

## 18. Критерии готовности MVP

Magic Action Builder готов, если:

- пользователь может выбрать систему;
- выбрать magic skill;
- выбрать базовый эффект;
- выбрать дополнительные эффекты;
- увидеть итоговую сложность;
- увидеть траты advantage/threat;
- скопировать результат;
- official content хранится только как safe summary + source/page ref.
