# Campaign Handbook / Content Packs

## 1. Назначение

**Campaign Handbook / Content Packs** — инструмент для мастера, который позволяет собрать разрешённый контент кампании в один понятный справочник для игроков.

Фича отвечает на вопросы:

- какие архетипы разрешены;
- какие карьеры разрешены;
- какие навыки используются;
- какие таланты доступны;
- какие предметы можно покупать;
- какая магия используется;
- какие героические способности разрешены;
- какие домашние правила действуют;
- где в книге искать нужное правило.

Это не замена книге. Это навигационный и организационный слой поверх разрешённого контента.

---

## 2. Пользовательская ценность

### Для мастера

Мастер может:

- собрать правила конкретной кампании;
- ограничить доступные опции;
- добавить домашние правила;
- дать игрокам один понятный справочник;
- не объяснять каждый раз, какие книги и опции разрешены;
- безопасно ссылаться на официальные источники без копирования больших текстов.

### Для игрока

Игрок может:

- открыть handbook кампании;
- понять, что разрешено;
- увидеть источники и страницы;
- использовать handbook при создании и развитии персонажа;
- видеть house rules мастера.

---

## 3. Основная концепция

Фича состоит из двух уровней:

1. **Content Pack** — набор разрешённого/запрещённого контента.
2. **Campaign Handbook** — опубликованное представление Content Pack для конкретной кампании.

Мастер создаёт Content Pack, наполняет его записями и привязывает к кампании.

Игроки видят Campaign Handbook в read-only режиме.

---

## 4. Типы записей

Content Pack может включать:

- Archetype;
- Career;
- Skill;
- Talent;
- Item;
- Heroic Ability;
- Spell;
- Magic Action;
- Alchemy Recipe;
- Rune;
- House Rule;
- Custom Note.

## 5. Сущности

## 5.1. ContentPack

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| OwnerUserId | Владелец |
| CampaignId | Кампания, если привязан |
| Name | Название |
| Description | Описание |
| System | Genesys Core / Realms of Terrinoth |
| IsPublicToCampaign | Видно ли игрокам |
| CreatedAt | Дата создания |
| UpdatedAt | Дата обновления |

## 5.2. ContentPackEntry

| Поле | Описание |
|---|---|
| Id | Идентификатор |
| ContentPackId | Pack |
| ContentType | Тип записи |
| ContentId | Ссылка на существующий справочник, если есть |
| Title | Название |
| AllowedState | Allowed / Disallowed / AskGM |
| SafeSummary | Краткое безопасное описание |
| Source | Источник |
| PageRef | Страница |
| GmNotes | Приватные заметки GM |
| PlayerNotes | Заметки для игроков |
| Tags | Теги |
| SortOrder | Порядок сортировки |

## 5.3. HouseRule

House Rule может быть отдельной записью ContentPackEntry.

Поля:

- title;
- category;
- rule text;
- examples;
- visibility;
- updatedAt.

Категории:

- Character Creation;
- Combat;
- Magic;
- Equipment;
- XP;
- Campaign Tone;
- Custom.

## 6. Основные сценарии

## 6.1. GM создаёт Content Pack

1. GM открывает кампанию.
2. Переходит во вкладку **Handbook**.
3. Нажимает **Создать Content Pack**.
4. Указывает название, систему и описание.
5. Сохраняет.

## 6.2. GM добавляет разрешённый контент

1. GM открывает Content Pack.
2. Нажимает **Добавить из справочника**.
3. Выбирает тип: talents, items, careers, heroic abilities, magic actions.
4. Использует фильтры: system, setting, tier, source, tags.
5. Выбирает несколько записей.
6. Нажимает **Добавить**.
7. Записи появляются в pack.

## 6.3. GM помечает контент

Каждая запись может иметь статус:

- **Allowed** — разрешено;
- **Disallowed** — запрещено;
- **Ask GM** — только с разрешения мастера.

Примеры:

- “Разрешены все таланты tier 1–2”;
- “Магия доступна только персонажам с карьерными magic skills”;
- “Огнестрельное оружие запрещено”;
- “Редкие предметы выше rarity 6 только с разрешения GM”.

## 6.4. GM добавляет house rule

1. GM нажимает **Добавить house rule**.
2. Заполняет название, категорию, текст, примеры и видимость.
3. Сохраняет.
4. Игроки видят правило в handbook, если оно опубликовано.

## 6.5. GM публикует handbook

1. GM включает **Visible to players**.
2. Игроки кампании получают доступ к read-only handbook.
3. GM может в любой момент скрыть handbook.

## 6.6. Игрок читает handbook

1. Игрок открывает кампанию.
2. Переходит во вкладку **Handbook**.
3. Видит разрешённые опции, запрещённые опции, Ask GM, house rules, source/page refs и заметки мастера для игроков.

## 7. Функции

## 7.1. Content Pack Manager

Функции:

- создать pack;
- редактировать;
- дублировать;
- удалить;
- назначить кампании;
- снять с кампании;
- опубликовать;
- скрыть.

## 7.2. Bulk Add

Мастер должен иметь возможность быстро добавлять много записей:

- все talents конкретного setting;
- все talents tier 1;
- все items типа weapon;
- все heroic abilities;
- все magic actions конкретного skill.

## 7.3. Поиск и фильтры

В handbook должны быть:

- поиск по названию;
- фильтр по типу;
- фильтр по статусу;
- фильтр по source;
- фильтр по tags.

## 7.4. Экспорт

Content Pack должен поддерживать:

- copy as markdown;
- print view;
- export JSON;
- import JSON.

Ограничение:

- export не должен содержать полный официальный текст из книг;
- только safe summary, source, page ref и custom notes.

## 9. UI

## 9.1. Вкладка Handbook

Внутри CampaignDetailView:

- если пользователь GM: manager mode, кнопки редактирования, publish toggle;
- если игрок: read-only view.

## 9.2. Player View

Игрок видит:

- название handbook;
- описание;
- список категорий;
- поиск;
- фильтры;
- карточки записей.

Карточка записи:

- название;
- тип;
- allowed state;
- safe summary;
- source;
- page ref;
- player notes.

## 9.3. GM View

GM видит всё, что игрок, плюс:

- кнопки редактирования;
- GM notes;
- bulk add;
- delete;
- reorder;
- publish/hide.

---

## 10. Юридическая безопасность

Для официального контента:

- не хранить длинные описания;
- не копировать текст книги;
- использовать short safe summary;
- обязательно указывать source;
- желательно указывать page ref;
- делать handbook помощником, а не заменой книги.

Для custom content:

- можно хранить полный пользовательский текст;
- нужно помечать как custom.

---

## 11. MVP

В MVP входит:

- создание Content Pack;
- записи;
- allowed/disallowed/ask GM;
- house rules;
- привязка к кампании;
- read-only player handbook;
- source/page refs;
- поиск и фильтры.

---

## 12. Не входит в MVP

- marketplace;
- публичные packs сообщества;
- автоматический legal review;
- полная интеграция с character creation;
- сложный import official books.

---

## 13. Критерии готовности

Фича готова, если:

- GM может создать Content Pack;
- GM может добавить записи;
- GM может добавить house rules;
- GM может назначить pack кампании;
- игрок кампании видит опубликованный Handbook;
- игрок не видит приватный Handbook;
- official entries не содержат длинный copyrighted text.
