# Encounter Builder

## 1. Назначение

**Encounter Builder** — инструмент подготовки сцен, столкновений и конфликтов для кампании Genesys / Realms of Terrinoth.

Фича позволяет мастеру заранее собрать:

- участников;
- противников;
- цели сцены;
- осложнения;
- заметки;
- награды;
- публичное описание;
- скрытую информацию;
- и затем одной кнопкой отправить всё в Game Table.

Encounter Builder не должен быть калькулятором баланса в стиле D&D. Genesys балансируется нарративно, поэтому фича помогает готовить структуру сцены, а не обещает точный CR.

---

## 2. Пользовательская ценность

### Для мастера

Мастер может:

- заранее подготовить боевые сцены;
- готовить социальные конфликты;
- готовить погони;
- готовить исследования;
- хранить энкаунтеры в кампании;
- быстро отправлять сцену в Game Table;
- печатать энкаунтер;
- повторно использовать шаблоны.

### Для игрока

Игрок видит только публичную часть энкаунтера, если мастер её открыл:

- название;
- player description;
- цели;
- публичных NPC;
- handouts.

---

## 3. Типы энкаунтеров

Поддерживаемые типы:

- Combat;
- Social;
- Exploration;
- Chase;
- Investigation;
- Travel;
- Hazard;
- Mixed;
- Custom.

Тип влияет на UI-подсказки, но не должен навязывать механику.

## 6. Основные сценарии

## 6.1. Создание энкаунтера

1. GM открывает кампанию.
2. Переходит во вкладку **Encounters**.
3. Нажимает **Создать энкаунтер**.
4. Заполняет название, тип, threat level, локацию, описание для мастера и описание для игроков.
5. Сохраняет.

## 6.2. Добавление NPC

1. GM открывает энкаунтер.
2. Нажимает **Добавить NPC**.
3. Выбирает NPC из Adversary Library.
4. Указывает количество, minion group/отдельные участники, hidden/visible и заметки.
5. Сохраняет.

## 6.3. Добавление персонажей кампании

GM может:

- добавить всех активных персонажей кампании;
- добавить выбранных персонажей;
- не добавлять PC заранее.

Кнопки:

- **Добавить всех PC**;
- **Выбрать PC**.

## 6.4. Описание целей сцены

Энкаунтер должен поддерживать цели, потому что не все сцены являются боем.

Примеры:

- “Остановить ритуал за 5 раундов”;
- “Убедить капитана открыть ворота”;
- “Задержать врагов до прибытия союзников”;
- “Поймать курьера”;
- “Не дать свидетелю сбежать”;
- “Украсть документы без тревоги”.

## 6.5. Осложнения

Осложнения — это свободные заметки мастера.

Примеры:

- плохая видимость;
- толпа мешает движению;
- горящая таверна;
- мост рушится;
- прибывает подкрепление;
- NPC не хочет убивать героев;
- магическое поле искажает заклинания;
- каждые 2 раунда растёт опасность.

## 6.6. Отправка в Game Table

1. GM открывает подготовленный энкаунтер.
2. Нажимает **Отправить в Game Table**.
3. Система проверяет активную сцену.
4. Если активной сцены нет — создаёт новую.
5. Если активная сцена есть — предлагает заменить текущую сцену, добавить участников к текущей сцене или отменить.
6. Система создаёт GameTableParticipants.
7. Система копирует notes и player description в сцену.
8. GM переходит на Game Table.

---

## 7. Видимость для игроков

Если `IsVisibleToPlayers = true`, игроки могут видеть:

- название;
- player description;
- player goals;
- публичных NPC;
- handouts;
- награды, если мастер открыл.

Игроки не видят:

- GM description;
- hidden participants;
- NPC goals;
- приватные complications;
- скрытые rewards;
- точные stats врагов, если GM не разрешил.

## 9. UI

## 9.1. Вкладка Encounters

Внутри CampaignDetailView:

- список энкаунтеров;
- кнопка создания;
- фильтры;
- поиск;
- карточки энкаунтеров.

Карточка показывает:

- название;
- тип;
- threat level;
- количество участников;
- видимость;
- теги;
- действия.

## 9.2. Encounter Editor

Разделы:

1. Основная информация.
2. Описание для мастера.
3. Описание для игроков.
4. Цели.
5. Участники.
6. Окружение и осложнения.
7. Награды.
8. Видимость.
9. Печать/экспорт.

## 10. MVP

В MVP входит:

- CRUD энкаунтеров;
- добавление NPC;
- добавление PC;
- ручной threat level;
- описание для GM;
- описание для игроков;
- осложнения;
- награды;
- visible/hidden;
- send to Game Table.

---

## 11. Не входит в MVP

- автоматический баланс;
- AI-генератор сцен;
- карты;
- токены;
- сложные триггеры фаз;
- real-time;
- marketplace энкаунтеров.

---

## 12. Критерии готовности

Фича готова, если:

- GM может создать энкаунтер;
- GM может добавить NPC и PC;
- GM может скрыть энкаунтер от игроков;
- GM может открыть публичную часть;
- GM может отправить энкаунтер в Game Table;
- игроки не видят приватные данные;
- энкаунтеры сохраняются в кампании.
