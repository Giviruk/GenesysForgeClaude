# Project Overview

## Назначение

GenesysForge — веб-приложение для создания и ведения листов персонажей для Genesys Core и Realms of Terrinoth. Продукт помогает игрокам вести персонажа, отслеживать XP, навыки, таланты, героические способности, инвентарь и автоматически пересчитывать производные характеристики.

Документ описывает не только идею, но и текущие границы реализации в репозитории.

## Целевая аудитория

- Игроки Genesys и Realms of Terrinoth.
- Ведущие, проверяющие корректность XP и прогресса персонажей.
- Игровые группы, использующие homebrew-контент.
- Разработчики и AI-агенты, поддерживающие проект.

## Основные пользовательские сценарии

- Зарегистрироваться или войти.
- Создать персонажа в одной из двух систем.
- Выбрать архетип/расу и карьеру.
- Выбрать бесплатные стартовые карьерные навыки.
- Повышать характеристики в фазе создания.
- Покупать и возвращать ранги навыков.
- Покупать и возвращать таланты с проверкой пирамиды.
- Назначить героическую способность для Realms of Terrinoth.
- Добавлять предметы, менять количество и состояние экипировки.
- Видеть пересчитанные wounds, strain, soak, defense, encumbrance.
- Создавать custom skills, talents, items, heroic abilities.
- Вести заметки персонажа.
- Создавать кампании, присоединять персонажей по join code и вести заметки кампании.
- Создавать NPC, энкаунтеры и активную сцену Game Table для кампании.
- Собирать магические действия и печатать карточки игровых материалов.

## Границы текущего MVP

Implemented:

- Личный аккаунт и JWT auth.
- Личные персонажи.
- Две системы: Genesys Core и Realms of Terrinoth.
- Базовые справочники через seed data.
- Creation phase и часть progression rules.
- Custom content, видимый только владельцу.
- Character notes.
- Campaigns, join code, membership and campaign notes.
- NPC/adversary library, encounter builder, Game Table and campaign content packs.
- Magic reference/action builder and browser-print cards for GM/player materials.
- Docker/CI/test foundation.

Partially implemented:

- UI покрывает основные операции; URL deep links есть для основных областей, но не для каждого под-экрана, и нет полноценного E2E покрытия.
- Talent effects автоматизированы только для числовых пассивных бонусов.
- Описания справочников есть, но требуют юридической осторожности.
- Печать реализована для карточек и GM-материалов через browser print; полного печатного листа персонажа пока нет.

Также реализовано (после первоначального MVP):

- Вход через Google (отключён до настройки `Auth:Google:ClientId`), refresh-токены с ротацией, self-service сброс пароля (e-mail — стаб в логе).
- URL deep links и real-time обновления кампаний/Game Table через SignalR.

Not implemented yet:

- Публичные shareable-ссылки на лист персонажа без авторизации.
- Импорт/экспорт персонажа.
- Полный PDF/print layout листа персонажа.
- Реальная отправка e-mail для сброса пароля.
- Механическое исполнение активных талантов, оружейных атак и heroic ability effects.

## Copyright ограничения

Проект не должен содержать оригинальные тексты из официальных книг. Запрещены копии описаний талантов, способностей, предметов, архетипов и карьер. Разрешены числовые параметры, названия, структурные правила, собственные краткие paraphrase-описания и пользовательский homebrew-контент.
