# GenesysForge Help

This section walks you through the basic scenario without the README: create a character, manage advancement, prepare a scene as the GM, and understand what data is shown in the different content modes.

GenesysForge does not replace the rulebooks. The app uses structural data, numeric parameters and short original descriptions, without any original text from the official books.

## Player quick start

- Open "Characters" and click "New character".
- Pick a system: Genesys Core or Realms of Terrinoth.
- Pick an archetype/species and a career.
- Mark your free career skills if the GM allows the standard starting set.
- Check the available XP and finish creation once the starting sheet is ready.

During creation some purchases can be undone. After creation is completed, refunds of starting purchases become unavailable: further advancement happens through XP awards and new purchases.

## Characteristics, skills and career skills

- Characteristics can only be bought during the creation phase.
- Skills are bought with XP. Career skills are cheaper, non-career skills cost more.
- Free career ranks gained at creation do not count as XP purchases and cannot be refunded.
- The dice pool is calculated automatically from the characteristic and the skill rank.

If a buy button is unavailable, the usual reasons are insufficient XP, the creation-phase limit, or the maximum rank rule.

## Talents and the pyramid

Talents are bought by tier. Higher tiers require maintaining the pyramid: there must be enough lower-tier talents to keep the upper level valid.

If a talent is ranked, buying it again counts as the next effective tier. If a talent is not ranked, it can only be bought once.

## Inventory and derived characteristics

- Add items on the "Inventory" tab.
- For armor and weapons the item state matters: "Equipped", "Carried" or "In backpack".
- Equipped armor affects soak and defense according to the app's rules.
- Encumbrance and the encumbrance threshold are recalculated automatically.
- Weapons in the inventory show the dice pool for the weapon's skill if the character has that skill.

If a derived characteristic did not change, check the item quantity and the "Equipped" state.

## Magic

The "Magic" page shows the actions available for the selected system. On the character sheet, the "Magic" tab uses the character's skills and shows their dice pool.

Pick a base magic action, add extra effects and check the resulting difficulty. The action card can be opened in print mode.

## Campaigns and joining players

The GM creates a campaign and gives the player a join code. The player joins the campaign with one of their characters. After that the GM can open the player's sheet from the campaign, and the player sees the campaign materials available to them.

## NPCs and the bestiary

In the "Bestiary" you can create NPCs, duplicate existing cards and use them in encounters. Duplication is handy for preparing variants of the same enemy without touching the original card.

NPCs have characteristics, skills, attacks, abilities, tags and visibility. Do not copy original book text into custom cards; use your own short descriptions.

## Encounters and the game table

- In a campaign, open "Encounters".
- Create an encounter, add NPCs and player characters.
- Set up the description visibility, each side's goals, the threat level and GM notes.
- Click "Send to game table" to move the participants into the active scene.

The game table keeps participants, initiative, story points, and public and GM notes. The GM runs the scene; players see the part available to them.

## PublicSafe and PrivateFull content modes

GenesysForge supports two modes of reference content.

- PublicSafe: a safe public set — names, numbers, structural parameters, sources and short paraphrase descriptions.
- PrivateFull: a private mode for local use where extended original materials of the project can be plugged in.

When preparing a public deployment, use PublicSafe and do not add original text from the official books to seeds, reference data or user documentation.

## Export, import and printing

A character sheet can be exported to JSON and imported back. This is convenient for backups or moving between instances.

The print sheet opens from the character card. In the browser you can print the page or save it as a PDF.
