# GenesysForge — Аудит реализации

Этот документ создан для Claude Code. Задача: сравнить текущее состояние репозитория с требованиями ниже и составить список того, что **реализовано**, что **частично реализовано** и что **отсутствует**.

---

## Как проводить аудит

1. Просканируй структуру репозитория (`tree` или `find`)
2. Для каждого раздела ниже — проверь наличие соответствующей логики в коде
3. Составь итоговый отчёт в формате таблицы: **Реализовано / Частично / Отсутствует**
4. Для отсутствующего — добавь краткую рекомендацию по реализации

## Контекст проекта

**Genesys Forge** — веб-приложение для создания и управления персонажами системы Genesys RPG (Fantasy Flight Games), с акцентом на сеттинг Realms of Terrinoth.

**Основные конкуренты:**
- Genesys Emporium (genesysemporium.com) — базовый билдер, v0.2.1.5, нет кастомного контента
- RPG Sessions (rpgsessions.com) — live play инструмент, слабо покрывает pre-play и homebrew
- GenesysRef (genesysref.netlify.app) — справочник, требует JSON для homebrew

**Незакрытые боли сообщества (приоритет при разработке):**
1. Кастомные виды / карьеры / таланты без правки кода
2. Terrinoth-специфичный контент (Heroic Abilities, расы, бестиарий)
3. Экспорт / импорт персонажей между инструментами
4. Нормальный PDF-экспорт листка персонажа
5. Генератор противников с балансировкой по Power Level
